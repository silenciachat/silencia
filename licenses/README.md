# Silencia License Files

This directory contains the full text of all licenses used in the Silencia project.

## License Files

- **AGPL-3.0.txt** - GNU Affero General Public License v3.0
  - Used for: Core library crates (`crates/silencia-*`)
  
- **MIT.txt** - MIT License
  - Used for: CLI application, examples
  
- **APACHE-2.0.txt** - Apache License 2.0
  - Used for: SDK, dual-licensed applications
  
- **CC-BY-4.0.txt** - Creative Commons Attribution 4.0 International
  - Used for: Documentation

## License Summary

### AGPL-3.0 (Core Libraries)
- **Strong copyleft**: Modifications must be open-sourced
- **Network use**: Must provide source code to network users
- **Best for**: Core cryptographic and networking libraries
- **Ensures**: Security through transparency

### MIT (CLI & Examples)
- **Permissive**: Can be used in proprietary software
- **Simple**: Short and easy to understand
- **Best for**: End-user applications and examples
- **Allows**: Commercial use without restrictions

### Apache-2.0 (SDK & Applications)
- **Permissive with patent grant**: Includes explicit patent license
- **Best for**: SDKs and enterprise integration
- **Provides**: Legal clarity for corporate users

### CC-BY-4.0 (Documentation)
- **Attribution required**: Must credit original authors
- **Best for**: Documentation and educational content
- **Allows**: Free sharing and adaptation

## Why Multiple Licenses?

Silencia uses different licenses for different components to balance:

1. **Security & Trust**: AGPL-3.0 for core crypto ensures transparency
2. **Adoption**: MIT/Apache-2.0 for apps enables broad use
3. **Knowledge Sharing**: CC-BY-4.0 for docs promotes education

This multi-license approach follows best practices for open-source cryptographic software.
